# MeshWorks Wiki – Contributor Guide

Минимальный набор знаний, чтобы править wiki.meshworks.ru без боли. Всё, что касается инфраструктуры, доступов и приватных runbook'ов, живёт локально и сюда не попадает.

## 1. Что редактируем
- Контент — папка `docs/` (Markdown/MDX). Каждый файл описывает страницу в навигации.
- Изображения — `static/img/wiki/`. Используем относительные пути `![](/img/wiki/<name>.png)`.
- Статические страницы вне docs (`/about` и подобные) лежат в `src/pages/`.
- Общие настройки и меню — `docusaurus.config.ts`, `sidebars.ts`, `src/css/custom.css`.

## 2. Быстрый старт
```bash
npm ci          # один раз
npm start       # dev-сервер на http://localhost:3000
```
Редактируйте файлы в `docs/**`, dev-сервер перезагрузится сам. Когда нужно проверить прод-версию — `npm run build && npm run serve`.

## 3. Требования к контенту
- Front matter: `title`, `slug`, `sidebar_label`, `sidebar_position`.
- Русский язык, нейтральный стиль, единицы измерения по SI. Код и команды — в бэктиках.
- Изображения заранее оптимизируем (ширина ≤1600px). Не коммитим тяжёлые бинарники.
- Внешние источники указываем в тексте или отдельным `source_url` по желанию.

## 4. Перед Pull Request
- `npm run check` (включает `tsc --noEmit` и `docusaurus build`).
- Убедитесь, что новые картинки лежат в `static/img/wiki/` и используются хотя бы в одном файле (`rg -l <name>`).
- Обновите README/CONTRIBUTING только если меняется процесс.
- В описании PR перечислите ключевые изменения и, по возможности, приложите скрин.

## 5. CI и деплой
- Каждое изменение проходит `./github/workflows/ci.yml`. Если он палится — смотрите лог в Actions.
- Деплой запускается вручную из GitHub Actions (`Deploy wiki`). Он синхронизирует репозиторий и перезапускает прод. Секреты и доступы хранятся в secrets, в публичном репозитории их нет.

## 6. Советы
- Не уверены, куда положить страницу? Сначала ищите похожий файл через `rg <тема> docs`.
- Для повторяющихся кусочков контента создавайте MDX-компоненты в `src/components/`.
- Если видите ошибку в структуре или стилях, заводите Issue — обсуждаем перед крупными правками.

На этом всё — никаких скрытых шагов. Ориентируйтесь на существующие страницы и PR-шаблон. EOF
